# 사용자 가이드

이 문서는 v-project 시스템을 일상 업무에서 사용하는 **일반 사용자**를 위한 안내서입니다. 로그인부터 알림 확인, 프로필 관리, Product Tour까지 자주 사용하는 기능을 시나리오 기반으로 설명합니다.

:::info 관리자용 기능을 찾고 계신가요?
사용자 관리, 권한 설정, 시스템 설정 등 관리자 전용 기능은 [관리자 가이드](../platform/admin-guide/admin-guide)를 참고하세요. 이 문서는 일반 사용자가 매일 사용하는 기능만 다룹니다.
:::

---

## 시나리오: 오늘 처음 가입한 홍길동씨의 하루

이해를 돕기 위해, 오늘 처음 v-project 시스템을 사용하게 된 "홍길동"씨의 하루를 따라가 보겠습니다. 각 섹션은 홍길동씨가 실제로 겪는 순서대로 구성되어 있습니다.

> **08:30** --- 관리자에게 받은 안내 메일을 확인하고 처음 로그인합니다  
> **08:35** --- Product Tour가 시작되어 화면 구조를 파악합니다  
> **08:40** --- 프로필 페이지에서 이름과 아바타를 설정합니다  
> **08:45** --- 임시 비밀번호를 본인만의 비밀번호로 변경합니다  
> **09:00** --- 대시보드에서 알림 벨을 눌러 공지사항을 확인합니다  
> **12:00** --- 점심시간에 다크 모드로 전환합니다  
> **17:00** --- 퇴근 전 로그아웃합니다

그럼 홍길동씨와 함께 시작해 볼까요?

## 1. 첫 로그인과 회원가입

### 시스템에 접속하기

브라우저(Chrome, Edge, Firefox 최신 버전 권장)를 열고 시스템 주소로 접속합니다.

```
http://127.0.0.1:5173
```

:::note 127.0.0.1을 사용하세요
`localhost` 대신 `127.0.0.1`을 사용해 주세요. WSL 환경에서 `localhost`는 IPv6 충돌로 접속이 안 될 수 있습니다.
:::

접속하면 로그인 화면이 표시됩니다. 화면 중앙에 로그인 폼이 있고, 그 아래에 SSO 버튼이 보일 수 있습니다.

### 이메일/비밀번호로 로그인하기

관리자가 계정을 미리 생성해 둔 경우, 안내받은 자격 증명으로 로그인합니다.

1. 로그인 화면 중앙의 **사용자명(또는 이메일)** 입력란에 관리자에게 받은 사용자명을 입력합니다.
2. 바로 아래 **비밀번호** 입력란에 임시 비밀번호를 입력합니다.
3. **"로그인"** 버튼을 클릭합니다.
4. 인증이 성공하면 시작 페이지(기본값: 대시보드)로 이동합니다.

:::tip 첫 로그인 직후에는 비밀번호를 꼭 변경하세요
관리자가 설정한 임시 비밀번호는 다른 사람이 알 수 있습니다. 로그인 후 가장 먼저 [비밀번호 변경](#비밀번호-변경하기)을 진행하는 것을 강력히 권장합니다.
:::

### Microsoft SSO로 로그인하기

조직에서 Microsoft Entra ID(구 Azure AD)를 사용하는 경우, 회사 계정으로 바로 로그인할 수 있습니다.

1. 로그인 화면 하단에 **"Microsoft로 로그인"** 버튼이 보입니다. 이 버튼을 클릭합니다.
2. Microsoft 로그인 페이지가 새 창 또는 같은 탭에서 열립니다.
3. 회사 이메일 주소와 비밀번호를 입력하고, 필요한 경우 MFA(다단계 인증)를 완료합니다.
4. 인증이 완료되면 자동으로 v-project 시스템으로 돌아오며 대시보드가 표시됩니다.
5. **처음 SSO로 로그인하면** 시스템 내 계정이 자동으로 생성됩니다. 별도의 회원가입 절차가 필요 없습니다.

### OIDC(범용 SSO)로 로그인하기

조직에서 Microsoft 외의 OIDC(OpenID Connect) 인증 서버를 사용하는 경우에도 SSO 로그인이 가능합니다. 로그인 화면에 해당 인증 서버 버튼이 표시되며, 클릭 후 외부 인증을 거쳐 동일하게 자동 로그인됩니다.

### 어떤 로그인 방식을 사용해야 하나요?

| 화면에 보이는 것 | 의미 | 권장 행동 |
|-----------------|------|----------|
| 로그인 폼만 보임 | 조직에서 ID/비밀번호 방식만 사용 | 관리자에게 받은 계정으로 로그인 |
| SSO 버튼도 보임 | 하이브리드 모드 (둘 다 사용 가능) | SSO 로그인 권장 (비밀번호 관리가 불필요) |
| SSO 버튼만 보임 | SSO 전용 모드 | SSO 버튼 클릭 |

### 비밀번호 요건

ID/비밀번호 방식으로 계정을 사용하는 경우, 비밀번호는 시스템 관리자가 설정한 정책을 충족해야 합니다. 일반적인 요건은 다음과 같습니다.

- 최소 길이 (보통 8자 이상)
- 영문 대문자, 소문자, 숫자, 특수문자 중 일정 조합 포함

비밀번호가 정책을 만족하지 않으면 화면에 구체적인 안내 메시지가 표시됩니다. 해당 메시지의 조건을 모두 충족하도록 비밀번호를 수정하면 됩니다.

:::warning 비밀번호 정책은 조직마다 다릅니다
위 요건은 일반적인 예시이며, 실제 요건은 관리자 설정에 따라 달라질 수 있습니다. 비밀번호 입력 시 화면에 표시되는 안내 메시지를 확인해 주세요.
:::

## 2. 프로필 설정하기

로그인에 성공하면 가장 먼저 프로필을 설정해 봅시다. 프로필 페이지에서는 이름, 아바타, 비밀번호 등을 관리할 수 있습니다.

### 프로필 페이지 열기

프로필 페이지로 이동하는 방법은 두 가지입니다.

**방법 1: 사이드바 메뉴 이용**
1. 화면 왼쪽의 사이드바에서 **"프로필"** 메뉴 항목을 찾아 클릭합니다.

**방법 2: 사이드바 하단 프로필 영역 이용**
1. 사이드바 최하단에 현재 로그인한 사용자의 이름과 아바타가 표시되어 있습니다.
2. 이 영역을 클릭하면 나타나는 메뉴에서 **"프로필"** 을 선택합니다.

### 이름 변경하기

1. 프로필 페이지 상단의 **기본 정보** 영역에서 **"이름"** 입력란을 찾습니다.
2. 원하는 이름(표시명)을 입력합니다. 이 이름은 사이드바, 알림, 감사 로그 등 시스템 전반에 표시됩니다.
3. 페이지 하단 또는 섹션 하단의 **"저장"** 버튼을 클릭합니다.
4. 화면 우측 상단에 "저장되었습니다" 같은 토스트 알림이 잠시 나타나면 변경이 완료된 것입니다.

### 아바타 변경하기

1. 프로필 페이지 상단에 현재 아바타(프로필 이미지)가 표시됩니다. 기본값은 이름의 첫 글자로 된 원형 아이콘입니다.
2. 아바타 이미지 영역을 클릭하거나 근처의 **"변경"** 버튼을 클릭합니다.
3. 파일 선택 대화상자가 열리면 원하는 이미지 파일을 선택합니다.
4. 이미지가 업로드되면 미리보기가 표시되고, **"저장"** 버튼을 클릭하여 반영합니다.

### 비밀번호 변경하기

:::note SSO 전용 계정은 이 기능을 사용할 수 없습니다
Microsoft SSO나 OIDC로만 로그인하는 경우, v-project 내에서 비밀번호를 관리하지 않습니다. 비밀번호 변경은 회사의 SSO 시스템(예: Microsoft 계정 설정)에서 진행해 주세요.
:::

1. 프로필 페이지에서 아래로 스크롤하여 **"비밀번호 변경"** 섹션을 찾습니다.
2. **현재 비밀번호** 입력란에 지금 사용 중인 비밀번호를 입력합니다.
3. **새 비밀번호** 입력란에 변경하고 싶은 비밀번호를 입력합니다.
4. **새 비밀번호 확인** 입력란에 같은 비밀번호를 한 번 더 입력합니다.
5. **"비밀번호 변경"** 버튼을 클릭합니다.
6. "비밀번호가 변경되었습니다"라는 확인 메시지가 표시되면 완료입니다.

변경에 실패한 경우 다음을 확인해 보세요.

| 오류 메시지 | 원인 및 해결 |
|------------|-------------|
| "현재 비밀번호가 일치하지 않습니다" | 현재 비밀번호를 정확히 입력했는지 확인하세요 |
| "비밀번호 정책을 충족하지 않습니다" | 최소 길이, 특수문자 등의 요건을 화면 안내에 맞게 수정하세요 |
| "새 비밀번호가 일치하지 않습니다" | 새 비밀번호와 확인 입력란에 같은 값을 입력했는지 확인하세요 |

## 3. 대시보드 둘러보기

프로필 설정을 마쳤다면, 이제 시스템의 전체 화면 구조를 살펴봅시다.

### 화면 구조 한눈에 보기

v-project의 모든 앱은 동일한 화면 구조를 공유합니다.

```
┌──────────────────────────────────────────────────────────┐
│  [배너 알림 영역] --- 관리자 공지가 있으면 여기 표시       │
├────────┬─────────────────────────────────────────────────┤
│        │  [콘텐츠 헤더] --- 현재 페이지 제목 + 액션 버튼   │
│ 사     │─────────────────────────────────────────────────│
│ 이     │                                                 │
│ 드     │                                                 │
│ 바     │              메인 콘텐츠 영역                     │
│        │         (각 페이지의 실제 내용)                   │
│ 메     │                                                 │
│ 뉴     │                                                 │
│        │                                                 │
│        │                                                 │
├────────┤                                                 │
│ 프로필  │                                                 │
│ 테마   │                                                 │
└────────┴─────────────────────────────────────────────────┘
```

### 좌측 사이드바

화면 왼쪽에 항상 표시되는 **좌측 사이드바**는 시스템의 핵심 네비게이션입니다.

- **상단 영역**: 앱 로고 또는 시스템 이름이 표시됩니다.
- **메뉴 영역**: 접근 가능한 페이지들이 아이콘과 텍스트로 나열됩니다. 본인의 권한에 따라 표시되는 메뉴 항목이 다릅니다. 예를 들어, "사용자 관리"나 "감사 로그" 같은 관리자 전용 메뉴는 일반 사용자에게는 보이지 않습니다.
- **하단 영역**: 현재 로그인한 사용자의 이름과 아바타가 표시됩니다. 이 영역을 클릭하면 프로필, 로그아웃 등의 메뉴가 나타납니다.
- **테마 전환 아이콘**: 사이드바 최하단에 있는 아이콘을 클릭하면 라이트/다크/시스템 테마를 전환할 수 있습니다.

### 콘텐츠 헤더

메인 콘텐츠 영역의 최상단에는 **콘텐츠 헤더**가 있습니다. 현재 페이지의 제목이 표시되고, 페이지에 따라 "새로 만들기", "내보내기" 같은 액션 버튼이 우측에 배치되기도 합니다.

### 알림 벨 아이콘

화면 우측 상단에 **종 모양 아이콘(알림 벨)**이 있습니다. 안 읽은 알림이 있으면 아이콘 위에 빨간색 숫자 배지가 표시됩니다. 자세한 내용은 [4. 알림 확인하기](#4-알림-확인하기) 섹션에서 다룹니다.

### 테마 전환하기 (라이트 / 다크 / 시스템)

홍길동씨처럼 점심시간에 다크 모드로 바꾸고 싶다면 아주 간단합니다.

1. 좌측 사이드바 최하단의 **테마 전환 아이콘**(해/달 모양)을 클릭합니다.
2. 클릭할 때마다 다음 순서로 전환됩니다.

| 모드 | 설명 | 아이콘 |
|------|------|--------|
| **라이트** | 밝은 배경에 어두운 텍스트 | 해 모양 |
| **다크** | 어두운 배경에 밝은 텍스트 | 달 모양 |
| **시스템** | 운영체제의 테마 설정을 자동으로 따름 | 모니터 모양 |

:::tip
시스템 모드를 선택하면 OS에서 다크 모드를 켜거나 끌 때 자동으로 따라가므로, 한 번 설정해 두면 별도 조작이 필요 없습니다.
:::

### 시작 페이지 변경하기

매번 로그인할 때마다 특정 페이지를 먼저 보고 싶다면, 시작 페이지를 변경할 수 있습니다.

1. 좌측 사이드바에서 **"프로필"** 메뉴를 클릭합니다.
2. 프로필 페이지 내의 **"세션 설정"** 또는 **"시작 페이지"** 영역을 찾습니다.
3. 드롭다운 목록에서 원하는 페이지를 선택합니다 (예: 대시보드, 통계 등).
4. **"저장"** 버튼을 클릭합니다.
5. 다음 로그인부터 선택한 페이지가 첫 화면으로 표시됩니다.

:::note
본인의 권한으로 접근 가능한 페이지만 시작 페이지로 지정할 수 있습니다. 시작 페이지 설정은 개인 설정이므로 다른 사용자에게 영향을 주지 않습니다.
:::

## 4. 알림 확인하기

v-project는 시스템 상태나 중요한 공지를 사용자에게 전달하기 위해 여러 종류의 알림을 제공합니다. 알림을 통해 시스템 점검, 새로운 공지, 권한 변경 등의 소식을 놓치지 않고 확인할 수 있습니다.

### 토스트 알림 (실시간 팝업)

화면 우측 상단에 잠시 나타났다가 자동으로 사라지는 알림입니다.

- **어디에 나타나나요?** 화면 우측 상단에 작은 카드 형태로 슬라이드되어 나타납니다.
- **언제 나타나나요?** 작업 완료, 오류 발생, 새 알림 수신 등 실시간 이벤트가 발생할 때마다 표시됩니다.
- **얼마나 보이나요?** 몇 초(보통 3~5초) 후 자동으로 사라집니다.
- **여러 개가 동시에 오면?** 위에서 아래로 쌓여 표시됩니다.
- WebSocket(실시간 통신)을 통해 전달되므로 페이지를 새로고침하지 않아도 수신됩니다.

### 상단 배너 알림 (공지)

관리자가 전체 사용자에게 전달하는 **공지 성격의 알림**입니다.

- **어디에 나타나나요?** 화면 최상단에 가로로 긴 띠(배너) 형태로 표시됩니다. 사이드바와 콘텐츠 영역 위에 걸쳐서 나타납니다.
- **언제 사용되나요?** 시스템 점검 안내, 중요한 업데이트 공지, 긴급 알림 등에 사용됩니다.
- **사라지지 않나요?** 관리자가 비활성화할 때까지 지속적으로 표시됩니다. 일부 배너는 사용자가 닫을 수 있고, 일부는 닫을 수 없습니다 (관리자 설정에 따라 다름).

### 안 읽은 알림 확인하기 (알림 벨)

1. 화면 우측 상단의 **종 모양 아이콘(알림 벨)**을 확인합니다.
   - 안 읽은 알림이 있으면 빨간색 숫자 배지가 아이콘 위에 표시됩니다 (예: 빨간 원 안에 "3").
   - 배지가 없으면 안 읽은 알림이 없다는 뜻입니다.
2. **종 모양 아이콘을 클릭합니다.**
3. 알림 목록이 드롭다운으로 펼쳐집니다. 각 알림에는 제목, 내용 요약, 수신 시간이 표시됩니다.
4. 개별 알림을 클릭하면 해당 알림의 상세 내용을 확인할 수 있으며, 자동으로 읽음 처리됩니다.
5. 드롭다운 상단 또는 하단의 **"모두 읽음 처리"** 버튼을 클릭하면 모든 안 읽은 알림을 한꺼번에 읽음 상태로 변경합니다.

### 브라우저 알림 허용하기

토스트 알림 외에, 브라우저의 푸시 알림을 허용하면 다른 탭에서 작업 중일 때도 데스크톱 알림을 받을 수 있습니다.

1. 시스템에 처음 접속하면 브라우저가 **"알림을 허용하시겠습니까?"** 라는 팝업을 표시할 수 있습니다.
2. **"허용"** 을 클릭하면 브라우저 알림이 활성화됩니다.
3. 이미 차단한 경우, 브라우저 주소창 왼쪽의 자물쇠 아이콘을 클릭하고 "알림" 항목을 "허용"으로 변경한 뒤 페이지를 새로고침하면 됩니다.

### 알림 관리하기

불필요한 알림이 너무 많다고 느끼시면 알림 유형별로 수신 여부를 조정할 수 있습니다.

1. 알림 드롭다운 하단의 **"알림 설정"** 링크를 클릭하거나, 프로필 페이지 내의 **"알림 설정"** 영역으로 이동합니다.
2. 각 알림 유형 옆의 **토글 스위치**를 켜거나 끕니다.
3. 변경 사항은 즉시 적용됩니다.

:::warning 시스템 알림은 끌 수 없습니다
관리자가 보내는 시스템 알림(공지, 긴급 안내 등)은 개인 설정으로 비활성화할 수 없습니다. 이는 중요한 정보를 모든 사용자에게 확실히 전달하기 위한 설계입니다.
:::

## 5. 비밀번호 변경 및 분실

### 비밀번호 변경 (로그인된 상태에서)

현재 비밀번호를 알고 있는 상태에서 변경하는 방법입니다.

1. 좌측 사이드바에서 **"프로필"** 메뉴를 클릭합니다.
2. 프로필 페이지 내의 **"비밀번호 변경"** 섹션으로 스크롤합니다.
3. **현재 비밀번호**, **새 비밀번호**, **새 비밀번호 확인**을 차례로 입력합니다.
4. **"비밀번호 변경"** 버튼을 클릭합니다.
5. 성공 메시지가 토스트로 표시됩니다.

### 비밀번호 분실 (Forgot Password)

비밀번호를 잊어버려서 로그인을 할 수 없는 경우, 이메일을 통해 재설정할 수 있습니다.

**단계별 절차:**

1. 로그인 화면에서 로그인 폼 하단의 **"비밀번호를 잊으셨나요?"** 링크를 클릭합니다.
2. 비밀번호 재설정 페이지가 열립니다. **이메일 주소** 입력란에 계정에 등록된 이메일을 입력합니다.
3. **"재설정 링크 보내기"** 버튼을 클릭합니다.
4. "재설정 메일이 발송되었습니다"라는 안내 메시지가 화면에 표시됩니다.
5. 이메일 받은편지함을 확인합니다. 발신자가 "v-project" 또는 시스템 이메일 주소인 메일을 찾습니다.
6. 메일 본문에 포함된 **"비밀번호 재설정"** 링크를 클릭합니다.
7. 브라우저에서 비밀번호 재설정 페이지가 열립니다.
8. **새 비밀번호**를 입력하고, **확인** 입력란에 같은 비밀번호를 한 번 더 입력합니다.
9. **"비밀번호 재설정"** 버튼을 클릭합니다.
10. "비밀번호가 변경되었습니다"라는 메시지와 함께 로그인 페이지로 이동합니다.
11. 새 비밀번호로 로그인합니다.

:::warning 재설정 메일이 도착하지 않는 경우
다음 사항을 확인해 보세요.

- **스팸/정크 메일함**을 확인하세요. 자동 발송 메일이 스팸으로 분류될 수 있습니다.
- 입력한 이메일이 **계정에 등록된 이메일과 정확히 일치**하는지 확인하세요.
- 5분 이상 기다려도 메일이 오지 않으면 **시스템 관리자에게 문의**하세요. 메일 서버(SMTP) 설정 문제일 수 있습니다.
- 개발 환경에서는 메일이 실제 이메일 주소가 아닌 MailHog(`http://127.0.0.1:8025`)로 발송됩니다.
:::

## 6. 세션 관리

### 세션이란?

로그인 후 시스템이 "이 사용자는 인증된 사람"이라고 기억하는 상태를 세션이라고 합니다. v-project는 JWT(JSON Web Token)라는 인증 토큰을 사용하여 세션을 관리합니다. 복잡한 원리를 이해할 필요는 없고, 아래의 실용적인 동작만 알아두시면 됩니다.

### 자동 로그아웃 (세션 타임아웃)

보안을 위해, 일정 시간 동안 시스템을 사용하지 않으면(마우스 클릭, 키보드 입력 등의 조작이 없으면) **세션이 자동으로 만료**됩니다.

- 세션이 만료된 후 어떤 버튼을 클릭하거나 페이지를 이동하면 **자동으로 로그인 화면으로 이동**합니다.
- 타임아웃 시간은 시스템 관리자가 설정합니다 (보통 30분~수 시간).
- 작성 중이던 폼 데이터는 저장되지 않을 수 있으므로, 중요한 작업은 **수시로 저장 버튼을 누르는 것**을 권장합니다.

### 토큰 만료 임박 시 동작

세션 만료 시간이 가까워지면 시스템이 자동으로 토큰을 갱신(refresh)하려고 시도합니다. 사용자가 활발하게 시스템을 사용하는 동안에는 세션이 자동으로 연장되므로, 정상적으로 사용 중인데 갑자기 로그아웃되는 일은 거의 없습니다.

다만, 다음과 같은 경우 토큰 갱신이 실패하여 로그인 화면으로 이동할 수 있습니다.

- 장시간 자리를 비운 경우 (타임아웃 초과)
- 네트워크 연결이 일시적으로 끊긴 경우
- 관리자가 해당 계정을 비활성화한 경우

### 다른 탭과의 동기화

같은 브라우저에서 v-project를 여러 탭으로 열어둔 경우, 세션 상태가 탭 간에 동기화됩니다.

- **한 탭에서 로그아웃하면 다른 탭도 함께 로그아웃됩니다.** 다른 탭에서 작업을 계속하려면 다시 로그인해야 합니다.
- 한 탭에서 세션이 만료되면 다른 탭에서도 동일하게 로그인 화면으로 이동합니다.
- 이는 보안을 위한 의도된 동작입니다.

:::tip 공용 PC를 사용하는 경우
공유 컴퓨터에서 작업한 후에는 반드시 **명시적으로 로그아웃**하세요. 브라우저를 닫아도 세션이 즉시 만료되지 않을 수 있습니다. 좌측 사이드바 하단의 프로필 영역을 클릭한 뒤 **"로그아웃"** 을 선택하면 확실하게 세션을 종료할 수 있습니다.
:::

### 로그아웃하기

1. 좌측 사이드바 최하단의 **사용자 프로필 영역**(이름과 아바타가 표시된 부분)을 클릭합니다.
2. 나타나는 메뉴에서 **"로그아웃"** 을 클릭합니다.
3. 로그인 화면으로 돌아갑니다. 세션이 완전히 종료되었습니다.

## 7. 도움말 및 Product Tour 이용법

v-project의 각 앱에는 기능별 도움말과 인터랙티브 가이드(Product Tour)가 내장되어 있습니다. 처음 사용하시는 분이라면 Product Tour를 통해 빠르게 화면 구조와 주요 기능을 익힐 수 있습니다.

### Product Tour란?

Product Tour는 화면의 주요 영역을 하나씩 강조(하이라이트)하면서 해당 영역의 기능을 설명해 주는 **단계별 인터랙티브 가이드**입니다. Driver.js 라이브러리를 기반으로 구현되어 있으며, 각 앱마다 맞춤화된 Tour 내용이 제공됩니다.

### 처음 접속 시 자동 시작

앱에 처음 접속하면 Product Tour가 **자동으로 시작**됩니다.

1. 화면이 살짝 어두워지면서 특정 영역(예: 사이드바)이 밝게 강조됩니다.
2. 강조된 영역 옆에 말풍선이 나타나며 해당 기능을 설명합니다.
3. 말풍선의 **"다음"** 버튼을 클릭하면 다음 영역으로 이동합니다.
4. 중간에 그만두고 싶으면 **"건너뛰기"** 버튼을 클릭하거나 키보드의 **Esc** 키를 누릅니다.
5. 마지막 단계에서 **"완료"** 버튼을 클릭하면 Tour가 종료됩니다.

:::note
Product Tour는 앱당 최초 접속 시 한 번만 자동으로 실행됩니다. 이후에는 직접 시작해야 합니다.
:::

### Product Tour 다시 보기

Tour를 다시 보고 싶거나, 처음에 건너뛰었는데 나중에 보고 싶을 때는 다음과 같이 합니다.

1. 화면 우측 상단의 **"?" 아이콘**(물음표 모양의 도움말 버튼)을 찾습니다.
2. 이 아이콘을 클릭합니다.
3. 나타나는 메뉴에서 **"가이드 투어 시작"** 또는 **"Tour 다시 보기"** 항목을 선택합니다.
4. Product Tour가 처음 단계부터 다시 시작됩니다.

:::tip 새로운 기능이 추가되었을 때
시스템이 업데이트된 후 Tour를 다시 실행하면 새로 추가되거나 변경된 UI 영역을 빠르게 파악할 수 있습니다. 업데이트 후 한 번쯤 Tour를 다시 실행해 보시는 것을 추천합니다.
:::

### 앱별 Tour 내용

각 앱은 자신만의 고유한 Tour 내용을 제공합니다. 예를 들어:

| 앱 | Tour에서 안내하는 주요 영역 |
|----|--------------------------|
| **v-channel-bridge** | 대시보드, 채널 라우팅 설정, 메시지 히스토리, 통계 차트 |
| **v-platform-template** | 기본 레이아웃, 사이드바 메뉴, 알림 벨, 프로필 설정 |
| **v-platform-portal** | 앱 카드, 앱 런처, 포털 네비게이션, 도움말 |

### 도움말 페이지

"?" 아이콘을 클릭하면 Tour 외에도 **도움말 페이지**로 이동할 수 있는 메뉴가 제공됩니다. 도움말 페이지에서는 해당 앱의 기능별 설명과 사용 팁을 텍스트 형태로 확인할 수 있습니다.

## 8. 앱 포털에서 앱 전환하기

v-platform-portal이 설정된 환경에서는 **한 번의 로그인으로 여러 앱을 자유롭게 이동**할 수 있습니다.

### 포털에서 앱으로 이동하기

1. 포털 주소(`http://127.0.0.1:5180`)에 접속하여 로그인합니다.
2. 포털 메인 화면에 등록된 앱들이 **카드 형태**로 표시됩니다. 각 카드에는 앱 이름, 설명, 현재 상태(온라인/오프라인)가 표시됩니다.
3. 이동하고 싶은 앱 카드를 **클릭**합니다 (예: "v-channel-bridge" 카드 클릭).
4. 별도의 로그인 없이 해당 앱의 대시보드가 열립니다. 이것이 **SSO Relay**입니다 --- 포털에서 1회용 코드를 발급하고, 앱이 이를 교환하여 자동 인증합니다.
5. 다른 앱으로 이동하려면 사이드바의 **"포털로 돌아가기"** 메뉴를 이용하거나, 브라우저에서 포털 주소로 직접 이동합니다.

:::note 오프라인 상태의 앱
앱 카드에 "오프라인"이라고 표시된 앱은 현재 서비스가 중단된 상태입니다. 클릭해도 접속할 수 없으며, 시스템 관리자에게 문의가 필요합니다.
:::

### 포털을 거치지 않고 직접 접속하기

각 앱의 주소로 직접 접속할 수도 있습니다. 이 경우 해당 앱의 로그인 화면이 표시되며, 별도로 로그인해야 합니다.

| 앱 | 직접 접속 주소 |
|----|--------------|
| v-channel-bridge | `http://127.0.0.1:5173` |
| v-platform-template | `http://127.0.0.1:5174` |
| v-platform-portal | `http://127.0.0.1:5180` |

## 9. 자주 묻는 질문 (FAQ)

### Q1. 로그인이 안 됩니다

**ID/비밀번호 로그인인 경우:**

- 사용자명과 비밀번호를 정확히 입력했는지 확인하세요. **대소문자를 구분**합니다.
- 키보드의 **Caps Lock**이 켜져 있지 않은지 확인하세요.
- 비밀번호를 잊어버렸다면 [비밀번호 분실](#비밀번호-분실-forgot-password) 절차를 따라 재설정하세요.
- 여러 번 틀리면 계정이 일시적으로 잠길 수 있습니다. 잠시 기다린 후 다시 시도하세요.
- 위 방법으로도 해결되지 않으면 **관리자에게 계정 상태 확인을 요청**하세요. 계정이 비활성화되었을 수 있습니다.

**SSO 로그인인 경우:**

- 회사 Microsoft 계정의 비밀번호가 만료되지 않았는지 확인하세요.
- SSO 버튼이 로그인 화면에 보이지 않는다면, 조직에서 SSO를 활성화하지 않은 것입니다. 관리자에게 문의하세요.
- Microsoft 로그인 페이지에서 오류가 발생하면 회사 IT 부서에 SSO 설정을 확인해 달라고 요청하세요.

---

### Q2. 알림이 안 옵니다

다음을 순서대로 확인해 보세요.

1. 화면 우측 상단의 **종 모양 아이콘**을 클릭하여 알림 목록을 직접 확인하세요. 토스트가 뜨지 않았을 뿐 알림은 와 있을 수 있습니다.
2. 알림 설정에서 특정 알림 유형을 **비활성화하지 않았는지** 확인하세요.
3. 브라우저의 **알림 차단 설정**을 확인하세요. 주소창 왼쪽 자물쇠 아이콘 → "알림" 항목이 "차단"으로 되어 있다면 "허용"으로 변경하세요.
4. 페이지를 **새로고침(F5)**하여 WebSocket(실시간 통신) 연결을 다시 수립해 보세요.
5. 위 방법으로도 해결되지 않으면 관리자에게 알림 서비스 상태를 확인해 달라고 요청하세요.

---

### Q3. 비밀번호 요건이 무엇인가요?

비밀번호 정책은 시스템 관리자가 설정하며, 조직마다 다릅니다. 일반적으로 다음 조건을 요구합니다.

- 최소 8자 이상
- 영문 대문자, 소문자, 숫자, 특수문자 중 일정 조합 포함

비밀번호를 입력할 때 화면에 **구체적인 요건이 실시간으로 표시**되므로, 해당 안내를 따르시면 됩니다. SSO 전용 계정은 v-project 내 비밀번호가 없으므로 해당 사항이 없습니다.

---

### Q4. 메뉴가 안 보입니다 (특정 페이지에 접근할 수 없습니다)

v-project는 **RBAC(역할 기반 접근 제어)** 시스템을 사용합니다. 사용자의 역할과 권한에 따라 볼 수 있는 메뉴가 다릅니다.

- 좌측 사이드바에는 **본인의 권한으로 접근 가능한 메뉴만** 표시됩니다.
- "사용자 관리", "감사 로그", "시스템 설정" 등은 관리자 전용 메뉴입니다.
- 업무상 특정 메뉴에 접근이 필요하다면 **관리자에게 권한 변경을 요청**하세요.
- 같은 팀 동료는 접근 가능한데 본인만 안 된다면, **권한 그룹** 소속 여부를 관리자에게 확인해 보세요.

---

### Q5. 다른 앱으로 이동했는데 다시 로그인하라고 합니다

- **포털을 통해** 앱에 접속했는지 확인하세요. 앱 주소로 직접 접속하면 SSO Relay가 동작하지 않아 별도 로그인이 필요합니다.
- 세션이 만료되었을 수 있습니다. **포털에 다시 로그인**한 후 앱으로 이동하세요.
- 1회용 SSO 코드는 30초 내 사용해야 합니다. 네트워크 지연으로 시간이 초과되면 포털에서 다시 앱 카드를 클릭하세요.

---

### Q6. 화면이 깨져 보이거나 레이아웃이 이상합니다

- **브라우저 캐시를 삭제**하고 강제 새로고침(`Ctrl + Shift + R`)해 보세요.
- **지원 브라우저**(Chrome, Edge, Firefox 최신 버전)를 사용하고 있는지 확인하세요. Internet Explorer는 지원하지 않습니다.
- 브라우저 확대/축소 비율이 **100%**인지 확인하세요 (`Ctrl + 0`으로 초기화).
- 다크 모드에서만 문제가 발생한다면 라이트 모드로 전환하여 동일한 문제가 재현되는지 확인하고, 관리자에게 보고하세요.

---

### Q7. Product Tour를 다시 보고 싶습니다

1. 화면 우측 상단의 **"?" 아이콘**(물음표 모양)을 클릭합니다.
2. **"가이드 투어 시작"** 또는 **"Tour 다시 보기"** 를 선택합니다.
3. Product Tour가 처음 단계부터 다시 시작됩니다.

자세한 내용은 [7. 도움말 및 Product Tour 이용법](#7-도움말-및-product-tour-이용법) 섹션을 참고하세요.

---

### Q8. 세션이 자주 만료됩니다

- 세션 타임아웃 시간은 시스템 관리자가 설정합니다. 타임아웃이 너무 짧게 느껴진다면 **관리자에게 연장을 요청**하세요.
- 시스템을 사용하는 동안(클릭, 입력 등의 조작)에는 세션이 자동으로 갱신됩니다. 아무 조작 없이 오래 방치한 경우에만 만료됩니다.
- **네트워크 연결이 불안정**한 경우에도 토큰 갱신이 실패하여 세션이 끊길 수 있습니다. Wi-Fi 또는 유선 연결 상태를 확인하세요.

---

### Q9. 다크 모드로 전환하고 싶습니다

좌측 사이드바 최하단의 **테마 전환 아이콘**(해/달 모양)을 클릭하세요. 클릭할 때마다 라이트 → 다크 → 시스템 순서로 전환됩니다. 자세한 내용은 [테마 전환하기](#테마-전환하기-라이트--다크--시스템) 섹션을 참고하세요.

---

### Q10. 포털에서 앱이 "오프라인"으로 표시됩니다

해당 앱의 서비스가 현재 중단된 상태입니다. 이런 경우에는:

- 잠시 기다린 후 페이지를 새로고침하여 상태가 변하는지 확인하세요.
- 지속적으로 오프라인이라면 **시스템 관리자에게 문의**하세요. Docker 컨테이너가 중지되었거나 서버에 문제가 발생한 것일 수 있습니다.
