# v-platform-portal 시작 가이드

이 문서를 읽고 나면 포털이 무엇인지 이해하고, Docker로 실행한 뒤 첫 로그인까지 완료할 수 있습니다. 또한 포털과 개별 앱 직접 접속의 차이를 파악하여 상황에 맞는 접근 방식을 선택할 수 있습니다.

## 포털이란

v-platform-portal은 v-project에 등록된 모든 앱(v-channel-bridge, v-platform-template 등)을 **한 화면에서 관리하고 원클릭으로 접속**할 수 있는 통합 런처입니다. 주요 역할은 다음과 같습니다.

- **앱 런처**: 등록된 앱을 카드 그리드 형태로 보여주고, 클릭 한 번으로 앱을 엽니다.
- **SSO Relay**: 포털에서 한 번 로그인하면 1회용 코드 기반으로 각 앱에 자동 인증되어 앱별 로그인이 필요 없습니다.
- **시스템 상태 모니터링**: 각 앱의 온라인/오프라인 상태와 응답 시간을 실시간으로 표시합니다.
- **사이트맵**: 모든 앱의 메뉴 구조를 한눈에 보여줍니다.
- **앱 관리 (CRUD)**: 관리자가 Admin UI에서 앱을 추가, 수정, 비활성화, 삭제합니다.

## 다른 앱과의 관계

포털은 v-platform 프레임워크 위에서 동작하는 **독립 앱** 중 하나입니다. 다른 앱(v-channel-bridge, v-platform-template)과 동일하게 `PlatformApp`을 사용하며, 플랫폼이 제공하는 로그인, 사용자 관리, 권한, 감사 로그, 설정 등의 공통 페이지를 모두 포함합니다.

```
v-platform (공통 프레임워크)
  |
  +-- v-platform-portal   ← 통합 런처 / SSO 허브 (이 앱)
  +-- v-channel-bridge     ← Slack/Teams 메시지 브리지
  +-- v-platform-template  ← 새 앱 시작 템플릿
```

핵심 차이는 포털이 **다른 앱들의 메타 정보(URL, 상태, 메뉴)를 수집하고 통합 진입점을 제공**한다는 것입니다. 각 앱은 포털 없이도 독립적으로 실행할 수 있습니다.

## Docker로 실행하기

포털은 Docker Compose의 `portal` 프로필로 실행합니다.

1. 프로젝트 루트 디렉토리(`v-project/`)에서 아래 명령을 실행합니다.

   ```bash
   docker compose --profile portal up -d --build
   ```

   다른 앱도 함께 실행하려면 프로필을 추가합니다.

   ```bash
   # 포털 + 템플릿 앱
   docker compose --profile portal --profile template up -d --build
   ```

2. 빌드가 완료될 때까지 기다립니다. 처음 실행 시 수 분이 소요될 수 있습니다.

3. 컨테이너 상태를 확인합니다.

   ```bash
   docker ps --filter name=v-platform-portal
   ```

   `v-platform-portal-backend`(포트 8080)과 `v-platform-portal-frontend`(포트 5180) 두 개의 컨테이너가 Running 상태여야 합니다.

## 접속 URL

| 서비스 | URL |
|--------|-----|
| 포털 프론트엔드 | `http://127.0.0.1:5180` |
| 포털 백엔드 API | `http://127.0.0.1:8080` |
| 백엔드 API 문서 (Swagger) | `http://127.0.0.1:8080/docs` |

## 첫 로그인 플로우

1. 브라우저에서 `http://127.0.0.1:5180`을 엽니다.
2. 로그인 화면이 나타납니다. 관리자가 발급한 계정 또는 SSO를 사용하여 로그인합니다.
3. 로그인에 성공하면 **앱 포탈** 메인 화면으로 이동합니다.

## 앱 런처 화면 구성

로그인 후 표시되는 메인 화면은 세 영역으로 나뉩니다.

1. **앱 카드 그리드**: 등록된 앱이 카드 형태로 나열됩니다. 각 카드에는 앱 이름, 설명, 아이콘, 온라인 상태 배지, 응답 시간이 표시됩니다. 카드를 클릭하면 해당 앱이 새 탭에서 열리며, JWT 토큰이 자동으로 전달됩니다.
2. **시스템 상태 테이블**: 모든 앱의 상태(Online/Offline/Degraded), 개별 서비스(DB, Redis 등) 상태, 응답 시간을 테이블로 보여줍니다. 상단의 "새로고침" 버튼으로 즉시 갱신할 수 있고, 30초마다 자동 갱신됩니다.
3. **사이트맵**: 각 앱에 등록된 메뉴 항목을 앱별로 카드에 모아 표시합니다. 전체 시스템의 메뉴 구조를 한눈에 파악할 수 있습니다.

## 포털을 쓸지, 앱을 직접 쓸지 선택하기

| 상황 | 추천 방식 |
|------|-----------|
| 여러 앱을 번갈아 사용하는 경우 | **포털 사용** -- 한 번 로그인으로 모든 앱에 SSO 접속 |
| 전체 시스템 상태를 모니터링해야 하는 경우 | **포털 사용** -- 헬스 체크와 사이트맵을 한곳에서 확인 |
| 관리자가 앱을 등록/관리해야 하는 경우 | **포털 사용** -- 앱 관리 CRUD는 포털에서만 제공 |
| 특정 앱 하나만 집중적으로 사용하는 경우 | **앱 직접 접속** -- 해당 앱의 URL로 직접 접속 |
| 포털 서비스가 내려간 경우 | **앱 직접 접속** -- 각 앱은 독립적으로 동작하므로 직접 접속 가능 |

포털과 개별 앱은 동일한 데이터베이스와 인증 체계를 공유하므로, 어떤 경로로 접속하든 동일한 사용자 계정과 권한이 적용됩니다.
