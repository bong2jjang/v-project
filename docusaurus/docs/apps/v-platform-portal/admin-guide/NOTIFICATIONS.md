# 알림 관리

이 문서를 읽고 나면 포털의 알림 관리 기능을 이해하고, 시스템/앱 알림을 생성하거나 관리할 수 있습니다. 배너와 팝업의 차이를 파악하고, 사용자에게 알림이 어떻게 보이는지 알 수 있습니다.

## 개요

포털은 `@v-platform/core`에서 제공하는 알림 시스템을 사용합니다. 관리자는 알림 관리 페이지에서 공지사항이나 시스템 알림을 생성하고, 대상 범위, 심각도, 전달 방식을 지정할 수 있습니다. 알림은 `app_id` 기반으로 격리되어, 포털에서 생성한 알림은 포털 사용자에게만 표시됩니다.

## 시스템 알림과 앱 알림

### 시스템 알림

플랫폼 차원에서 자동 생성되는 알림입니다. 예를 들어 비밀번호 만료 예고, 보안 정책 변경 안내 등이 해당합니다. 시스템 알림은 관리자가 직접 편집하거나 삭제할 수 없으며, active 토글로 표시 여부만 제어할 수 있습니다.

### 앱 알림 (커스텀 알림)

관리자가 직접 작성하는 알림입니다. 공지사항, 점검 안내, 새 기능 소개 등 자유롭게 내용을 작성할 수 있습니다.

## 알림 관리 페이지 접근

1. 포털에 관리자 계정으로 로그인합니다.
2. 왼쪽 사이드바에서 **관리** 섹션의 **알림 관리** 메뉴를 클릭합니다.
3. 등록된 알림 목록이 표시됩니다.

## 알림 생성

1. 알림 관리 페이지 상단의 **새 알림** 버튼을 클릭합니다.
2. 알림 생성 모달이 열립니다. 아래 항목을 입력합니다.

### 입력 필드

| 필드 | 설명 |
|------|------|
| **제목** | 알림 제목. 사용자에게 가장 먼저 보이는 텍스트. |
| **내용** | 알림 본문. Markdown 형식 지원. 미리보기로 렌더링 결과를 확인할 수 있음. |
| **심각도** | 긴급, 오류, 경고, 정보, 성공 중 선택. 심각도에 따라 알림 아이콘과 색상이 달라짐. |
| **범위 (scope)** | **전역**: 모든 사용자 / **이 앱**: 현재 앱(포털) 사용자만 / **역할**: 특정 역할의 사용자만 / **사용자**: 특정 사용자만 |
| **전달 방식** | **토스트**: 화면 우측 하단에 일시적으로 팝업 / **공지사항**: 페이지 상단 배너로 표시 / **토스트 + 공지사항**: 양쪽 모두 표시 |

3. 입력을 완료한 뒤 **저장** 버튼을 클릭합니다.

## 알림 수정

1. 알림 목록에서 수정할 알림의 **수정** 버튼을 클릭합니다.
2. 모달에서 내용을 변경한 뒤 **저장** 버튼을 클릭합니다.
3. 시스템 알림은 수정 버튼이 표시되지 않습니다.

## 알림 삭제

1. 알림 목록에서 삭제할 알림의 **삭제** 버튼을 클릭합니다.
2. 확인 후 삭제됩니다. 시스템 알림은 삭제 버튼이 표시되지 않습니다.

## active 토글

각 알림에는 활성(active) 토글이 있습니다. 토글을 끄면 해당 알림이 사용자에게 더 이상 표시되지 않습니다. 시스템 알림도 토글로 숨길 수 있습니다.

- **토글 켜짐**: 알림이 사용자에게 전달됨
- **토글 꺼짐**: 알림이 숨겨짐 (삭제되지는 않음)

## 사용자에게 보이는 방식

### 토스트 알림

화면 우측 하단에 일시적으로 나타나는 팝업입니다. 몇 초 후 자동으로 사라집니다. 사용자가 놓치기 쉬우므로 간단한 안내나 낮은 심각도 알림에 적합합니다.

### 공지사항 배너

페이지 상단에 고정 배너로 표시됩니다. 사용자가 직접 닫기 전까지 계속 보입니다. 중요한 공지사항이나 시스템 점검 안내에 적합합니다.

### 알림 벨

화면 상단 우측의 알림 벨 아이콘에 읽지 않은 알림 수가 배지로 표시됩니다. 벨을 클릭하면 알림 팝오버가 열리며 최근 알림 목록을 확인할 수 있습니다.

## app_id 기반 격리

포털에서 생성한 알림의 범위를 "이 앱"으로 설정하면, 해당 알림은 포털(`v-platform-portal`)에서만 표시됩니다. 다른 앱(v-channel-bridge 등)에서는 보이지 않습니다.

마찬가지로 다른 앱에서 생성한 "이 앱" 범위의 알림은 포털에 표시되지 않습니다. "전역" 범위로 설정한 알림만 모든 앱에서 공통으로 표시됩니다.
