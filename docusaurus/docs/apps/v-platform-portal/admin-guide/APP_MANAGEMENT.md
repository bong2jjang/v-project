# 앱 관리

이 문서를 읽고 나면 관리자 권한으로 포털에 앱을 등록, 수정, 비활성화, 삭제할 수 있습니다. 각 필드의 의미를 이해하고, 환경변수 시드와 Admin UI CRUD의 관계를 파악할 수 있습니다.

## 사전 조건

- **system_admin** 역할을 가진 계정으로 포털에 로그인되어 있어야 합니다.
- 앱 관리 페이지는 관리자 전용이며, 일반 사용자에게는 사이드바 메뉴에 표시되지 않습니다.

## 앱 관리 페이지 접근

1. 포털(`http://127.0.0.1:5180`)에 관리자 계정으로 로그인합니다.
2. 왼쪽 사이드바에서 **관리** 섹션의 **앱 관리** 메뉴를 클릭합니다.
3. 등록된 앱 목록이 카드 그리드로 표시됩니다. 비활성 앱도 포함하여 모든 앱이 보입니다.

## 앱 등록

1. 앱 관리 페이지 상단의 **앱 등록** 버튼을 클릭합니다.
2. "앱 등록" 모달이 열립니다. 아래 필드를 입력합니다.
3. 모든 필수 항목을 입력한 뒤 **등록** 버튼을 클릭합니다.
4. 등록에 성공하면 모달이 닫히고, 앱 관리 목록에 새 카드가 즉시 추가됩니다.
5. 포털 메인 화면(앱 포탈)의 앱 런처에도 즉시 반영됩니다.

### 필드 설명

| 필드 | 필수 | 설명 | 예시 |
|------|------|------|------|
| **앱 ID** (`app_id`) | 필수 | 시스템 내부 식별자. 소문자, 숫자, 하이픈만 허용. 등록 후 변경 불가. | `v-channel-bridge` |
| **표시 이름** (`display_name`) | 필수 | 사용자에게 보이는 앱 이름. | `Channel Bridge` |
| **설명** (`description`) | 선택 | 앱에 대한 간단한 설명. 앱 카드에 표시됨. | `Slack/Teams 메시지 브리지` |
| **아이콘** (`icon`) | 선택 | Lucide 아이콘 이름. 아이콘 선택기에서 클릭하여 지정. 기본값 `Box`. | `MessageSquare` |
| **프론트엔드 URL** (`frontend_url`) | 필수 | 앱 프론트엔드 접속 URL. 사용자가 카드를 클릭했을 때 열리는 주소. | `http://127.0.0.1:5173` |
| **API URL** (`api_url`) | 필수 | 앱 백엔드 API URL. 포털이 헬스 체크와 사이트맵 수집에 사용. Docker 내부 네트워크 주소를 권장. | `http://v-channel-bridge-backend:8000` |
| **헬스 엔드포인트** (`health_endpoint`) | 선택 | 헬스 체크 경로. 기본값 `/api/health`. | `/api/health` |
| **정렬 순서** (`sort_order`) | 선택 | 앱 런처에서의 표시 순서. 숫자가 작을수록 먼저 표시. 기본값 `0`. | `10` |
| **활성 상태** (`is_active`) | 선택 | 토글로 활성/비활성 전환. 비활성 앱은 일반 사용자 앱 런처에 표시되지 않음. 기본값 활성. | `true` |

### 앱 ID 규칙

- 소문자 영문, 숫자, 하이픈(`-`)만 사용 가능합니다. 정규식: `^[a-z0-9-]+$`
- 시스템 전체에서 고유해야 합니다. 이미 존재하는 `app_id`로 등록하면 `409 Conflict` 오류가 발생합니다.
- 한번 등록한 `app_id`는 수정할 수 없습니다. 변경이 필요하면 삭제 후 재등록해야 합니다.

## 앱 수정

1. 앱 관리 페이지에서 수정할 앱 카드의 **수정** 버튼을 클릭합니다.
2. "앱 수정" 모달이 열립니다. `app_id`를 제외한 모든 필드를 변경할 수 있습니다.
3. 변경 사항을 입력한 뒤 **수정** 버튼을 클릭합니다.
4. 수정에 성공하면 앱 관리 목록과 앱 런처에 변경 내용이 즉시 반영됩니다.

## 앱 비활성화

앱을 완전히 삭제하지 않고 일시적으로 숨기고 싶을 때는 비활성화를 사용합니다.

1. 앱 관리 페이지에서 해당 앱 카드의 **수정** 버튼을 클릭합니다.
2. 모달 하단의 **활성 상태** 토글을 꺼짐(비활성)으로 전환합니다.
3. **수정** 버튼을 클릭합니다.
4. 비활성화된 앱은 앱 관리 페이지에서 "비활성" 배지와 함께 계속 표시되지만, 일반 사용자의 앱 런처에서는 보이지 않습니다.

## 앱 삭제

1. 앱 관리 페이지에서 삭제할 앱 카드의 **삭제** 버튼을 클릭합니다.
2. 확인 모달이 나타납니다. 앱 이름과 ID를 확인합니다.
3. **삭제** 버튼을 클릭합니다.
4. 삭제된 앱은 데이터베이스에서 완전히 제거되며 복원할 수 없습니다. 앱 런처에서도 즉시 사라집니다.

### 삭제 vs 비활성화 선택 기준

| 상황 | 추천 |
|------|------|
| 앱 점검이나 일시적 중단 | **비활성화** -- 나중에 토글만 켜면 복원됨 |
| 더 이상 사용하지 않는 앱 | **삭제** -- 포털에서 완전히 제거 |
| 테스트용 앱 정리 | **삭제** -- 불필요한 항목 정리 |
| 앱 URL이 변경될 예정 | **비활성화** 후 URL 수정 후 다시 **활성화** |

삭제는 포털의 앱 레지스트리에서만 항목을 제거합니다. 실제 앱 서비스(Docker 컨테이너)에는 영향을 주지 않습니다.

## PORTAL_APPS 환경변수와 Admin UI의 관계

포털은 두 가지 방식으로 앱 정보를 관리합니다.

### 환경변수 시드 (PORTAL_APPS)

`docker-compose.yml`에서 `PORTAL_APPS` 환경변수를 설정하면, 포털이 최초 시작될 때 데이터베이스의 `portal_apps` 테이블이 **비어 있는 경우에만** 환경변수 값으로 초기 데이터를 삽입합니다. 이것은 일회성 시드(seed)입니다.

환경변수 형식은 파이프(`|`) 구분자로 앱 정보를 나열합니다.

```
PORTAL_APPS=app-id|표시이름|설명|아이콘|프론트엔드URL|APIURL
```

여러 앱은 쉼표(`,`)로 구분합니다.

### Admin UI CRUD (주된 관리 방식)

초기 시드 이후의 모든 앱 관리는 **포털 Admin UI(앱 관리 페이지)**에서 수행합니다. Admin UI에서 등록/수정/삭제한 내용은 데이터베이스에 저장되며, 인메모리 레지스트리에도 즉시 반영됩니다.

데이터베이스에 이미 앱이 존재하면 `PORTAL_APPS` 환경변수는 무시됩니다. 따라서 운영 환경에서는 환경변수를 설정할 필요 없이 Admin UI만으로 앱을 관리할 수 있습니다.

### 정리

| 시점 | 데이터 소스 | 비고 |
|------|-------------|------|
| 최초 실행, DB 비어 있음 | `PORTAL_APPS` 환경변수 | 일회성 시드 |
| 시드 이후 | 데이터베이스 (`portal_apps` 테이블) | Admin UI에서 CRUD |
| 포털 재시작 시 | 데이터베이스에서 로드 | 환경변수 무시 (DB에 데이터 존재) |
