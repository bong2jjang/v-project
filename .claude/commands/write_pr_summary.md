# PR 요약 작성

현재 브랜치의 변경 사항을 분석하여 PR 요약을 작성합니다.

## 워크플로우

1. `git merge-base HEAD main`으로 기준 커밋 확인
2. `git diff`로 전체 변경 사항 확인
3. `git log --oneline`로 커밋 메시지 확인
4. 변경 영역 분류: backend(API/모델/인증), frontend(페이지/컴포넌트/디자인시스템), docker, docs

## 출력 형식

```md
## 변경 사항 요약

[변경 사항에 대한 간결한 요약]

## 변경 타입

- [ ] 새로운 기능 (feat)
- [ ] 버그 수정 (fix)
- [ ] 리팩토링 (refactor)
- [ ] 문서 업데이트 (docs)

## 테스트 계획

[테스트 방법 요약]

## 관련 이슈

[관련 이슈 번호]
```

불릿 포인트는 적게, 이탤릭/볼드 과용 금지. 짧은 문장 사용.
